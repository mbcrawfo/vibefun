---
name: create-pr
description: Create a PR, wait for CI and reviews.
---

1. Create a PR with the current branch.
2. Watch the PR until you see all GitHub workflows complete.
3. If the CI workflow fails, investigate and correct the problems.
4. Look for code reviews and comments from CodeRabbit. If no reviews or comments appear, report the problem.
5. Carefully consider any suggestions from the comments and reviews. Make changes where necessary to address feedback. If you do not believe a comment is actionable, post a reply explaining your reasoning.
6. Always wait for CodeRabbit to finish its review before pushing new changes. After you push, repeat steps 2–6 to address any follow-up comments or subsequent workflow failures.
